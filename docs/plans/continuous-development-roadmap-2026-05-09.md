# 连续开发路线图（2026-05-09）

## 目的

本路线图用于承接当前核心循环原型，避免从“能跑”直接跳到长期系统导致断点。后续开发必须先完成当前阶段收尾，再推进正式可玩所需的核心系统，最后再扩展长期内容。

## 本次沟通结论

当前游戏方向没有跑偏。原始设计中的“装备 Build 爽感 + 放置轻节奏”仍然成立。

当前最强体验支点是：

```text
自动/手动战斗 -> 掉落 -> 过滤/入包/转化 -> 对比穿戴 -> DPS/EHP/评分提升 -> 推进层数 -> 离线收益
```

下一阶段设计主线收敛为：

```text
构筑选择 -> 推层目标 -> 掉落反馈
```

该主线第一版已经落地，详见 `../records/build-progression-feedback-review.md`。P0 稳定化已经完成，详见 `../records/p0-stabilization-review-2026-05-11.md`。现阶段不要继续横向堆强化、套装、转生、多职业、排行榜等长期系统。当前应先推进内容厚度、真实构筑入口和经济掉落调优。

## 阶段 0：当前原型收尾

目标：把已经完成的核心闭环整理成稳定、可继续扩展的基线。

必须完成：

- 保持 `npm run format:check`、`npm run typecheck`、`npm run test:run`、`npm run build` 全部通过。
- 每次阶段性开发结束后更新 `docs/backlog/remaining-work-2026-05-09.md` 和相关记录。
- 清理临时计划，只保留复盘、总任务拆解、当前设计方向和路线图。
- 确认存档导入导出不会恢复自动挂机、战斗日志、离线弹窗等运行态。
- 保持组件边界：组件展示，composables 编排，stores 改状态，core 做纯逻辑。

不在此阶段做：

- 新长期系统。
- 大规模 UI 改版。
- 正式美术批量接入。

## 阶段 1：正式可玩必备核心系统

目标：让游戏从“核心循环能跑”进入“玩家可以稳定游玩并理解目标”的状态。

### 1. 构筑评分系统

问题：当前装备评分是通用分，会把玩家引导成只穿最高分，流派差异不足。

必须完成：

- 新增评分模式：`balanced`、`crit`、`speed`、`tank`。
- 保留现有通用评分作为 `balanced`。
- 在设置或角色状态中保存当前评分偏好。
- 让装备对比、更优高亮、只看更优、分解保护使用同一评分模式。
- 补核心测试，验证同一批装备在不同评分模式下排序不同。

验收标准：

- 暴击词缀装备在 `crit` 模式下更容易被判定为更优。
- 攻速词缀装备在 `speed` 模式下更容易被判定为更优。
- 生命、护甲、抗性装备在 `tank` 模式下更容易被判定为更优。
- 玩家能明确知道当前系统按什么偏好推荐装备。

### 2. 推层目标系统

问题：当前层数推进偏后台逻辑，玩家还不清楚适合挂机哪里、为什么失败、为什么收益降低。

必须完成：

- 推荐挂机层。
- 当前挑战层和最高解锁层的关系展示。
- 战斗失败原因提示。
- 收益倍率原因说明。
- 推层成功反馈。

验收标准：

- 玩家能看到“当前建议挂机第几层”。
- 玩家失败后能知道是输出不足、生存不足还是战力偏低。
- 玩家推层成功后能感知到新层数解锁。

### 3. 掉落反馈系统

问题：当前掉落主要依赖日志和文字，缺少关键奖励反馈。

必须完成：

- 传说装备掉落提示。
- 离线领取成功反馈。
- 背包满和离线收益截断提示强化。
- 层数突破提示。

验收标准：

- 关键奖励不只出现在日志里。
- 离线回来能清楚知道本次最重要的收益。
- 背包满造成损失时，玩家知道该整理背包。

### 4. 基础内容差异

问题：怪物和层数差异仍较弱，持续推层的内容感不足。

必须完成：

- 每 10 层轻量 Boss 标记。
- Boss 推荐战力更高，奖励或掉落略有差异。
- 不同阶段怪物名称和基础属性有明显变化。

验收标准：

- 玩家能感到 10 层一个小目标。
- Boss 层能形成短期目标，但不要求复杂机制。

### 5. 稳定性和测试

问题：当前测试集中在 core 和 stores，组件交互还没有覆盖。

必须完成：

- 引入组件交互测试策略。
- 优先覆盖装备对比、分解确认、筛选面板、存档导入确认。
- 对新增构筑评分和推荐层逻辑补核心测试。

验收标准：

- 核心评分和推荐层逻辑有纯函数测试。
- 关键弹窗不会因重构失效。

## 阶段 2：正式可玩增强

目标：在核心目标清晰后，增强留存和反馈，但仍不铺开长期大系统。

可做内容：

- 简化版天赋系统：先 3 条路线，每条少量被动节点。
- 简化强化系统：只开放 +1 到 +5，失败无惩罚或低惩罚。
- 传说装备第一版：少量固定特殊词缀，不做大规模套装。
- 离线开箱表现：先做领取反馈和高品质优先展示，再做完整动画。
- 资源路径常量和占位目录：建立正式资源替换边界。

暂不做：

- 转生。
- 套装。
- 多职业。
- 排行榜。
- 复杂 Boss 技能。
- 服务端经济校验。

## 阶段 3：长期扩展补充

目标：在核心可玩且稳定后，再补长期目标和内容深度。

可扩展方向：

- 完整天赋树。
- 装备强化 +10 以上和高等级风险机制。
- 传说特效扩展。
- 远古装备。
- 套装收集。
- 转生或轮回。
- 多职业。
- 排行榜和装备展示。
- 正式音效系统。
- 完整掉落、开箱、强化和突破动效。
- 更严格的离线收益防作弊策略。

## 设计红线

- 不让背包整理变成主玩法。
- 不让通用装备评分成为唯一答案。
- 不在构筑目标明确前扩展大量长期系统。
- 不用正式资源替换掩盖玩法目标不清的问题。
- 不牺牲核心逻辑可测试性。

## 下一步执行建议

当前优先做 P1 内容厚度：

- 保持验证基线通过，并同步 backlog、阶段复盘和项目状态报告。
- 增加普通怪类型：高血怪、高攻怪、均衡怪、奖励怪。
- 强化第 10、20、30 层 Boss 的奖励倾向和阶段感。
- 增加金币层、经验层、装备层、Boss 层等关卡标签，并让推荐挂机说明引用这些标签。
- P1 完成后再进入小型天赋节点和 `goldFind`、`magicFind` 调优。

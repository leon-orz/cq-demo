## P0：下一阶段最优先

下一阶段主线应围绕：

```text
构筑选择 -> 推层目标 -> 掉落反馈
```

详见 `../design/current-design-direction-2026-05-09.md`。

### 装备流派评分权重

当前“更优装备”、自动分解保护和只看更优仍基于通用评分，不能准确覆盖暴击、攻速、坦克和主属性构筑。

建议拆分：

- 新增评分模式类型：`balanced`、`crit`、`speed`、`tank`、`mainAttribute`。
- 保留现有通用评分作为 `balanced` 默认模式。
- 将评分偏好保存到 `settings` 或 `player`。
- 让 `calculateItemScore`、更优判断、装备对比、只看更优和分解保护使用同一评分模式。
- 补核心测试：同一批装备在不同评分模式下排序应不同。

## P1：体验和维护增强

### 装备资源占位目录和路径常量

文档已有资源清单，但代码层还没有统一的资源路径入口。

建议拆分：

- 建立 `public/assets/items/`、`public/assets/ui/` 等占位目录。
- 新增资源路径常量，例如 `src/utils/assets.ts`。
- 给装备槽、品质边框、存档按钮、离线报告等当前文字占位点标注替换路径。
- 不引入未完成美术，只先建立可替换边界。

### 组件交互测试

当前测试主要覆盖 core 和 store，组件交互还没有引入组件测试依赖。

建议优先覆盖：

- 装备对比弹窗。
- 分解确认弹窗。
- 背包筛选面板。
- 存档导入覆盖确认。

### 离线收益表现增强

当前离线收益功能可用，但表现仍是静态报告。

建议拆分：

- 领取反馈。
- 开箱动画。
- 传说装备优先展示。
- 背包满损失提示强化。

## 长期风险

- 装备评分如果不按构筑调整，会导致更优高亮和自动分解不可信。
- 本地时间可被修改，当前离线收益只通过负数归零和上限封顶规避。
- 背包容量扩大后可能需要虚拟滚动或更严格的列表性能控制。
- 组件交互缺少测试，弹窗和筛选抽屉仍主要依赖人工验证。

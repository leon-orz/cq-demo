## 复盘方式

本轮由主 agent 统筹，子 agent 并行协作：

- 文档差距复核：对照 `docs/START_HERE.md` 与 `docs/DEVELOPMENT_ROADMAP.md`，确认阶段一剩余重点。
- UI/Store 风险复核：检查交互补强 diff，指出响应式对比和切层提示的提交前风险。
- 数值调校：围绕前 10 层强度、掉落率、训练和强化成本做保守调整。

主 agent 负责接管未收尾的调校结果、修复风险、补测试、写记录和提交准备。

- 离线收益已有测试保护，但仍建议用真实关闭、等待、重开流程再确认一次。
- 当前数值只是阶段一保守调校，后续应基于实际试玩记录继续微调掉落率、金币训练和强化消耗。

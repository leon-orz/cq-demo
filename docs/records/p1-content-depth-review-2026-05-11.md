# P1 内容厚度阶段复盘（2026-05-11）

## 目标回顾

本阶段目标是在不扩展大系统的前提下，让层数、怪物、Boss 和推荐挂机说明更有辨识度。重点服务当前闭环：

```text
构筑评分 -> 装备选择 -> 推层目标 -> 战斗/挂机 -> 掉落反馈 -> 离线收益 -> 存档
```

## 实际完成

- 新增怪物类型：均衡怪、高血怪、高攻怪、奖励怪和首领。
- 普通层按层数轮换怪物类型，怪物类型会影响生命、攻击、推荐战力和收益倾向。
- 新增关卡标签：金币层、经验层、装备层、Boss 层。
- 第 10、20、30 层 Boss 分别偏金币、经验、装备，30 层后按该模式循环。
- `StageTargetEvaluation` 新增怪物类型、标签、收益倾向和推荐原因。
- 推荐挂机卡片改为展示 `recommendReason`，不再只展示收益倍率文本。
- 中间战斗面板展示当前层标签、怪物类型和怪物占位符号。
- 核心测试覆盖普通怪轮换、Boss 奖励倾向、关卡标签和推荐说明。

## 主要取舍

- 怪物和关卡差异仍从层数派生，没有建立独立地图或关卡配置表，保持实现轻量。
- 关卡标签只服务解释和推荐说明，不扩展复杂地图词缀、抗性或事件系统。
- 奖励怪初版偏装备收益，金币和经验倾向主要交给 Boss 阶段感承载。
- UI 只做信息补充，没有加入新动画和新资源需求。

## 剩余风险

- 奖励怪和装备 Boss 可能让推荐挂机长期偏向装备层，P3 做 `magicFind` 和掉落调优时需要重新校准。
- 高攻怪会放大低生存构筑的失败风险，后续加入防御流天赋后应复核推层节奏。
- 当前关卡差异仍是公式派生，若后续需要章节主题、怪物池或手工关卡，应迁移到集中关卡配置。

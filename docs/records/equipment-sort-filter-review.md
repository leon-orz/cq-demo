# 装备排序和筛选面板阶段复盘模板

## 完成情况

本阶段当前处于文档规划状态。开发完成后在此记录实际实现范围、未完成项和偏离计划的原因。

## 预期验证能力

- 装备列表排序不会改变背包实体数据。
- 品质、部位、更优装备、锁定状态筛选可以独立启用，也可以组合使用。
- 升序和降序切换结果稳定。
- 筛选结果为空时，玩家能明确看到空结果状态并一键重置。
- 启用筛选后，背包入口或筛选按钮有可识别角标。

## 待复盘问题

- 排序字段是否覆盖玩家真实整理背包的高频需求。
- 筛选面板在移动端是否会遮挡关键操作。
- 筛选后执行分解时，玩家是否能理解最终分解范围仍以确认弹窗为准。
- “更优装备”基于通用评分是否足够可信，是否需要按流派权重拆分。
- 已启用筛选角标、推荐装备光晕和锁定角标是否存在视觉冲突。

## 实现取舍记录

开发完成后补充：

- 状态放在背包 store 还是单独 UI store。
- 默认排序字段和重置筛选后的排序行为。
- 是否保留当前背包滚动位置。
- 是否引入新的组件测试依赖。

## 风险

- 多条件筛选可能让玩家误以为装备消失，需要明显的筛选启用提示。
- 排序和筛选若直接写在组件中，后续自动分解、推荐装备和离线收益展示会重复实现。
- 推荐光晕和更优图标都表示提升，需要避免信息重复或含义冲突。

## 下一步建议

1. 开发前先固化排序字段、筛选字段和默认行为。
2. 优先补核心纯函数测试，再接入 Store 和组件。
3. 美术资源未完成前使用 CSS 占位，但保留稳定的资源替换命名。
4. 功能完成后回填本复盘，记录真实验证命令和残余风险。

# 装备排序和筛选面板阶段复盘

## 完成情况

本阶段已经完成背包排序和筛选面板的最小可行版本。排序筛选逻辑被放在 `core/item/inventoryView.ts`，背包视图状态被放在独立的 `stores/inventoryView.ts`，没有混入拾取过滤规则，也没有改变背包实体数据顺序。

## 已验证能力

- 装备列表可按评分、品质、部位、装备等级和新获得顺序排序。
- 排序不会改变 `inventory.items` 原始数组顺序。
- 品质、部位、最低装备等级、更优装备、隐藏锁定装备可以组合筛选。
- 筛选结果为空时展示明确空状态，并提供重置筛选入口。
- 启用筛选后，背包整理区域显示“已启用”状态。
- `RightPanel.vue` 不再直接调用 `getInventoryViewItems()`，可见列表、空文案和重置入口显示判断由 `inventoryView` store 提供。
- 新增 `tests/core/inventory-view.test.ts` 和 `tests/stores/inventory-view.test.ts`。

## 实现取舍

- 背包视图状态独立为 `inventoryView` store，避免与“拾取过滤”混淆。
- 背包展示派生数据由 store 对外提供，后续右侧面板拆分已复用该边界。
- UI 暂用 CSS 和文字占位，未接入正式图标、底板、空结果插画和光效。
- 筛选面板采用移动端可点击的底部抽屉/窄屏弹层，不依赖悬停。
- 初版不做搜索框、虚拟滚动和批量标签，等背包容量扩大后再评估。
- “更优装备”继续复用通用装备评分，后续需要按流派权重优化。

## 验证命令

- `npm run format:check`
- `npm run typecheck`
- `npm run test:run`
- `npm run build`

## 风险

- 筛选后执行分解时，玩家可能误以为只会分解当前可见装备；当前仍以分解确认弹窗的最终列表为准。
- 通用评分模型无法准确覆盖所有构筑，后续需要流派权重。
- 正式美术资源接入后，需要检查筛选角标、锁定角标和更优高亮是否冲突。

## 下一步建议

1. 接入正式图标、品质边框、筛选空结果插画和推荐装备光晕。
2. 后续引入组件测试后，补筛选抽屉和空结果重置的交互测试。

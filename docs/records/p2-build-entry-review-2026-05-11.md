# P2 真实构筑入口阶段复盘（2026-05-11）

## 目标回顾

本阶段目标是让构筑选择不再只停留在装备评分偏好，而是通过小型天赋节点真实影响角色属性、战斗表现或收益属性。

第一版坚持小型、配置化、可测试，不做复杂技能树。

## 实际完成

- 新增 `src/data/skills.ts`，配置暴击、攻速、防御、寻宝四条分支。
- 每条分支 3 个节点，共 12 个节点。
- `SkillNode` 扩展了分支和描述字段。
- `player` store 新增天赋点、已用点、可用点 getter。
- 天赋点由等级派生，当前规则为 `level - 1`。
- 新增激活节点、重置节点和节点归一化动作。
- 激活暴击、攻速、防御节点后会进入 `calculateTotalStats()`，真实影响 DPS 或 EHP。
- 激活寻宝节点后会进入总属性，提供 `goldFind` 和 `magicFind` 入口。
- 左侧角色面板新增收益属性展示：金币获取、魔法发现。
- 新增基础天赋 UI：`SkillPanel.vue`、`SkillNode.vue`，采用分支列表，不做复杂树状图。
- 补充核心、store 和组件测试。

## 主要取舍

- 没有做复杂技能树连线、前置节点、重置消耗和职业分支。
- 寻宝节点只让 `goldFind`、`magicFind` 进入总属性，暂不影响金币收益和掉落权重。
- 天赋点由等级派生，不新增单独存档字段，旧存档高等级角色会自然获得可用点数。
- 当前 `skillNodes` 仍随玩家状态保存，配置归一化负责补齐新增节点和同步节点数值。

## 剩余风险

- 天赋数值只是第一版保守值，后续需要结合 P3 经济和掉落调优重新校准。
- `goldFind`、`magicFind` 尚未影响实际金币收益和掉落权重，寻宝流暂时只完成入口，不完成完整闭环。
- 当前天赋节点没有前置关系，后续如果扩成完整技能树，需要把配置拆分得更细。
- 左侧面板内容开始变长，后续若继续增加角色系统，可能需要折叠或分页。

## 下一步建议

进入 P3：经济和掉落调优。

- 先让 `goldFind` 影响在线战斗和离线收益金币。
- 再让 `magicFind` 影响掉落率或高品质权重。
- 建立集中数值调参表，管理怪物成长、金币经验曲线、掉落率和掉落价值。
- 重新校准推荐挂机评分，确保金币层、经验层、装备层和寻宝构筑口径一致。

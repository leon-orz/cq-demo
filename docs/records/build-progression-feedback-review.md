# 构筑评分、推层目标与掉落反馈阶段复盘

## 完成范围

本阶段围绕“构筑选择 -> 推层目标 -> 掉落反馈”完成当前核心闭环收尾。

- 构筑评分系统：新增 `balanced`、`crit`、`speed`、`tank`、`mainAttribute` 评分模式。
- 评分偏好接入：装备对比、更优高亮、只看更优、分解保护和背包评分排序使用同一评分模式。
- 推层目标系统：新增推荐挂机层、推层目标、当前层评估、失败原因、收益倍率和击杀预估。
- 推层解锁规则：最高已解锁层真实战斗胜利后解锁下一层。
- 掉落反馈系统：高价值掉落、被过滤高价值装备、背包满、推层成功和离线领取都有全局反馈。
- 组件交互测试：引入 `@vue/test-utils`，覆盖背包工具栏、离线报告、奖励反馈层和战斗中心面板。

## 当前验证基线

- `npm run format:check`
- `npm run typecheck`
- `npm run test:run`
- `npm run build`

当前测试结果为 23 个测试文件、88 条测试通过。

## 主要取舍

- `item.score` 保留为 `balanced` 旧缓存，非均衡评分实时计算，兼容旧装备和存档。
- 推层推荐按已解锁层的 `gold/s + exp/s` 选择，不先纳入掉落期望。
- 掉落反馈先做全局提示层，不先做战斗画面内飘字或完整开箱动画。
- 组件测试先覆盖当前闭环关键交互，不把战斗数值和背包规则重复放进组件测试。

## 剩余风险

- 推荐挂机层尚未纳入掉落价值、Boss 奖励差异和高品质掉落期望。
- `progression.ts` 当前仍包含少量中文展示文案，后续文案复杂后可拆到展示层或 composable。
- `combat.playerPower` 和 `lastRewardMultiplier` 仍作为历史状态保留，UI 已主要读取 `progressionSummary`。
- 掉落反馈还没有自动消失和战斗区内动画，当前以手动关闭的全局反馈为主。

## 下一步建议

优先进入当前阶段收尾和轻量体验增强：

- 清理临时接续计划，保持文档入口清晰。
- 评估是否拆分推层文案和清理旧战力状态。
- 为掉落反馈增加自动消失或战斗区短反馈。
- 再推进每 10 层轻量 Boss、基础内容差异、资源路径常量和占位目录。

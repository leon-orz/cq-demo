# 装备对比与分解确认阶段复盘

## 完成情况

本阶段完成了正式装备对比弹窗和分解确认弹窗，替换了原生确认框，移动端也可以通过点击打开，不依赖悬停。

## 已验证能力

- 空装备槽位时，新装备被视为提升。
- 装备对比能返回属性差值列表。
- 戒指对比会选择评分较低的已穿戴戒指作为目标。
- 分解确认能展示保护原因。
- `ItemSlot.vue` 和 `ItemCompareModal.vue` 不再直接调用 `@/core/item/filter`，装备评分、评分差、更优判断和对比结果由 `useItemPresentation` 提供。

## 实现取舍

- 弹窗状态暂放在 `ItemSlot.vue` 和 `RightPanel.vue` 局部状态中。
- 装备展示派生数据先放在 composable 中，暂不新增 `inventoryUi` store。
- 暂不引入 `@vue/test-utils`，组件交互主要通过类型检查和核心/store 测试保障。
- 分解确认弹窗暂不展示装备图标，后续美术资源补齐后再加。

## 风险

- 每个 `ItemSlot` 自己管理对比弹窗，后续如果多入口打开对比，建议迁移到 `inventoryUi` store。
- 对比结果基于通用评分，后续需要按流派权重优化。
- 分解预览保护原因只覆盖当前三类规则，未来规则增加时需要同步扩展。

## 下一步建议

1. 优化离线奖励流程，让过滤规则先于背包容量截断。
2. 引入正式装备图标、品质边框、锁定图标和推荐装备光晕。
3. 后续增加组件测试依赖后，补弹窗交互测试。
4. 评估是否把弹窗打开状态迁移到独立 UI store，方便后续多入口复用。

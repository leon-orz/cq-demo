# 自动挂机与收益衰减开发记录

## 目标

本阶段补齐放置游戏最核心的在线挂机基础能力：

- 自动挂机开关。
- 定时触发战斗模拟。
- 根据战力与推荐战力计算收益衰减。
- 背包满时停止收益并暂停挂机。
- UI 展示当前战力、收益倍率、挂机次数和暂停原因。

## 任务拆解

主智能体负责集成实现、测试和记录。

子智能体 A 负责复核当前代码切入点：

- `src/core/combat/`
- `src/stores/combat.ts`
- `src/stores/inventory.ts`
- `src/components/layout/CenterPanel.vue`
- `tests/core/`、`tests/stores/`

子智能体 B 负责整理美术资源需求：

- 自动挂机状态反馈。
- 战斗区怪物与背景。
- 掉落与品质光效。
- 背包满提示。
- 后续离线开箱资源。

## 当前占位策略

战斗区暂时使用文字“魔”和 CSS 光效作为怪物占位，不引入真实图片。后续美术资源补齐后，应替换为怪物图、战斗背景、血条和状态图标。

## 验收标准

- 点击“自动挂机”后可持续触发战斗。
- 点击“停止挂机”后定时器停止。
- 背包满时不能继续手动挑战，并自动停止挂机。
- 战力不足时金币和经验按档位衰减。
- 类型检查、测试、构建通过。

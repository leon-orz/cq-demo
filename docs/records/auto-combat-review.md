# 自动挂机阶段复盘

## 完成情况

本阶段完成了在线自动挂机的第一版基础能力，核心行为已经从“手动点击挑战”扩展到“可启动、可停止、可因背包满或失败自动暂停”。

## 实现取舍

- 收益衰减使用 `DPS * 4 + EHP * 0.35` 估算当前战力，后续可以替换为更细的构筑评分。
- 自动挂机暂时使用固定间隔 `AUTO_COMBAT_INTERVAL_MS`，没有根据攻击速度或击杀时间动态调整。
- 战斗区只使用 CSS 和文字占位，避免在美术资源未齐时引入临时图片。
- 背包满时直接暂停挂机，符合设计文档中“背包满停止收益”的方向。

## 风险与后续优化

- 当前自动挂机没有离线收益，下一阶段应接入 `core/offline/reward.ts` 和存档时间戳。
- 当前收益衰减只影响金币和经验，装备掉落率暂未受衰减影响。
- 背包满提示仍是文本，后续需要接入图标和更明确的操作引导。
- 自动挂机定时器依赖前端页面存活，切后台后的行为需要与离线系统统一。

## 建议下一步

1. 实现离线收益结算和离线报告。
2. 增加自动拾取过滤，减少背包满频率。
3. 为装备掉落补品质高亮和简单掉落反馈。
4. 增加背包满后的“一键分解并继续挂机”操作。

# 离线收益阶段复盘

## 完成情况

本阶段已经完成离线收益的最小可行闭环：记录最后活跃时间、启动时计算离线收益、展示报告、玩家领取后入账。

## 已验证能力

- 离线时间上限裁剪。
- 战斗失败时无收益。
- 背包剩余格不足时装备截断。
- 收益倍率影响金币和经验。
- 领取报告后金币、经验、装备入账。
- 领取后报告清空，避免重复领取。

## 实现取舍

- 暂用 Pinia persistedstate 默认本地存储，未接入 localForage 快照。
- 离线批量模拟复用 `simulateBatchCombat()`，保留其 10000 次击杀保护上限。
- 离线报告先不做开箱动画，只做弹窗和列表。
- 当前离线报告不长期持久化，避免大量装备对象常驻存档。

## 风险

- 本地时间可被修改，当前只通过负数归零和 8 小时上限规避。
- 离线收益中装备掉落率暂未随收益衰减调整。
- `simulateBatchCombat()` 后续高层或高攻速情况下可能需要更高效的聚合公式。
- 刷新时如果离线时间很短，不弹报告，这是设计取舍，但需要后续结合自动保存频率再校准。

## 下一步建议

1. 做自动拾取过滤和装备分解保护，降低背包满导致离线收益截断的频率。
2. 增加装备对比和更优装备高亮，让离线回归后的整理更高效。
3. 给离线报告增加基础开箱动画和奖励领取反馈。
4. 后续统一设计完整存档系统，再把快照存储迁移到 localForage。

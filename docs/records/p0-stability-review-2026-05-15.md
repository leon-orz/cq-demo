## 复盘方式

本轮由主 agent 统筹，三个子 agent 并行处理互不重叠的稳定性问题：

- 战斗与经济：统一战斗耗时模型，修正 `magicFind` 单位和战斗日志收益。
- 装备一致性：修复重复穿戴已装备物品导致复制的风险。
- 存档可靠性：补即时保存、存档结构校验和坏档保护。

主 agent 负责集成、修复测试断言、全量验证、记录和提交准备。

## 完成内容

- 战斗时间模型改为基于实际战斗耗时推进：
  - `CombatEngine.simulateBatch()` 使用 `killTime/survivalTime` 推进时间。
  - 自动挂机用 `setTimeout` 按本场预计战斗耗时调度下一场。
  - 离线收益逐场扣除实际战斗耗时，胜利才结算，失败停止。
- 经济单位修正：
  - `magicFind` 与 `goldFind`、`expFind` 统一为小数百分比，`0.1` 表示 +10%。
  - 战斗日志显示含金币/经验获取加成后的实际发放值。
- 装备状态一致性修正：
  - 重复穿戴当前已装备物品时 no-op，不再把同一物品推回背包。
  - 一键穿戴候选去重，避免已装备物品参与重复替换。
  - 装备对比面板对当前已装备物品显示“已装备”，不提供替换按钮。
- 存档可靠性提升：
  - `pagehide` 与 `visibilitychange` 触发即时保存。
  - `useGameSave()` 提供 `cleanup()`，测试和组件卸载时可清理监听和防抖保存。
  - `SaveManager.load/importSave()` 增加基础结构校验，坏 JSON、缺字段、旧版本、高版本不会写入或污染状态。
  - 保存时同步 `lastOnlineTimestamp`，并作为离线时间 fallback。

## 验证结果

已通过：

- `npm run format:check`
- `npm run typecheck`
- `npm run test:run`：10 个测试文件，28 个测试通过
- `npm run build`

新增/扩展测试覆盖：

- 慢速击杀不会在短离线时长内刷出大量击杀。
- `simulateBatch()` 按实际战斗耗时推进。
- `magicFind` 按小数百分比提升掉落判定。
- 背包满暂停仍不发奖励。
- 重复穿戴已装备物品不会复制到背包或重复叠加属性。
- 一键穿戴不会产生重复装备。
- 坏档、旧版本、高版本导入失败且不污染状态。
- 页面隐藏和 `pagehide` 会即时保存，清理后不再触发。
- 重复恢复自动挂机不会重复启动战斗计时器。

## 剩余风险

- 真实浏览器长时间挂机尚未手工验收，尤其是实际节奏、金币/经验增长和掉落体感。
- `SaveManager` 结构校验仍是基础级，未完整校验每件装备和每条词缀的字段合法性。
- 离线收益按固定楼层重复模拟，暂未实现失败回退或逐层策略。
- 移动端三栏布局只做了基础响应式，仍需真实设备检查文本溢出和操作密度。

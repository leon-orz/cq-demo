## 主要取舍

- 训练第一版只做金币消耗和基础属性成长，不做完整强化、失败率、材料链、重置消耗或复杂训练树。
- `magicFind` 品质权重只做小幅加成并封顶，避免寻宝流过早压过战斗构筑。
- 背包压力只作为 UI 提示，不改变推荐挂机层评分，避免推荐口径混乱。
- Boss 仍不做复杂技能机制，先通过目标卡、奖励倾向和反馈增强阶段感。
- 本轮使用两个只读子 agent 梳理训练、推荐/Boss/背包接入边界，最终代码由主线程统一集成，避免跨公式口径漂移。

## 下一步建议

下一次优先做数值和体验校准，不急着扩新系统：

1. 使用干净存档或整理背包验证在线掉落链路，重点看入包、过滤、自动转化、高价值掉落反馈和 Boss 反馈。
2. 观察训练金币消耗是否足够吸收 `goldFind` 产出，必要时调整训练消耗曲线。
3. 观察 `magicFind` 稀有和传说产出以及背包压力，必要时调整品质权重封顶或拾取过滤默认值。
4. 若继续扩训练或天赋，优先考虑独立构筑页面，避免左侧面板继续膨胀。

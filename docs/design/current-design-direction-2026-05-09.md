# 当前阶段游戏设计方向（2026-05-09）

## 结论

原始设计的大方向仍然成立：本项目不是实时操作 ARPG，而是“装备 Build 爽感 + 放置轻节奏”的 H5 放置型 ARPG。当前核心循环已经跑通，下一阶段不应继续横向堆强化、套装、转生、多职业等长期系统，而应收敛到三件事：

```text
构筑选择 -> 推层目标 -> 掉落反馈
```

连续开发安排详见 `../plans/continuous-development-roadmap-2026-05-09.md`。

这三件事决定玩家为什么要整理装备、为什么要变强、为什么要再次上线。

## 应继续坚持的设计支柱

### 1. 装备驱动，而不是操作驱动

玩家价值应体现在选择装备、调整构筑和决定挂机层，而不是实时操作。战斗继续保持纯数值模拟，前端负责清晰展示结果。

继续坚持：

- 装备品质、部位、词缀、评分和对比。
- 自动战斗和离线收益。
- 背包自动过滤、一键分解、锁定保护和更优提示。
- 核心逻辑可测试，UI 不承载数值规则。

### 2. 离线即进度

离线收益是放置体验的关键支柱。当前收益结算、报告、领取和背包截断已经成立，下一步应补情绪反馈，而不是先增加更复杂的离线规则。

继续坚持：

- 8 小时左右收益上限。
- 背包空间影响收益。
- 离线报告清晰告知获得了什么、损失了什么。

### 3. 整理装备要高效

背包整理是体验入口，但不能变成负担。当前背包工具已经比较完整，后续应减少管理压力，强化“发现好装备”的爽点。

继续坚持：

- 自动过滤。
- 只看更优。
- 排序筛选。
- 分解保护。
- 装备对比。

## 当前需要调优的方向

### 1. 装备评分不能再只有通用分

当前通用评分已经承担了太多职责：

- 更优高亮。
- 装备对比。
- 分解保护。
- 背包筛选。

如果继续只用通用评分，玩家会变成“只穿最高分”，暴击、攻速、坦克和主属性构筑都会被压扁。下一步应优先建立流派评分权重。

建议第一版只做 4 个模式：

- `balanced`：当前默认通用评分。
- `crit`：重视暴击率、暴击伤害、攻击。
- `speed`：重视攻速、攻击、命中后收益类词缀。
- `tank`：重视生命、护甲、抗性和闪避。

主属性方向可以先作为权重修正，不急着做完整职业系统。

### 2. 玩家需要明确“为什么要变强”

当前能挑战、能挂机、能解锁层数，但推层目标仍偏后台数值。下一阶段应把层数目标变成玩家可理解的决策。

建议补：

- 推荐挂机层。
- 当前挑战层。
- 失败原因。
- 预计收益倍率。
- 推层成功后的明确奖励或解锁提示。

暂不需要复杂 Boss 系统，但每 10 层可以先有轻量 Boss 标记、名称和更高推荐战力。

### 3. 掉落反馈应少量高价值增强

不建议现在全面接正式美术资源。更合理的是先补少数关键反馈点：

- 传说装备掉落提示。
- 离线领取完成反馈。
- 背包满损失提示强化。
- 层数突破提示。

这些反馈直接服务核心循环，比大规模 UI 换皮更有价值。

## 应后置或简化的设计

### 暂不推进

- 大型天赋树。
- 装备强化 +10 完整曲线。
- 套装。
- 远古装备完整玩法。
- 转生。
- 多职业。
- 排行榜。
- 全量正式美术替换。
- 全套 GSAP 和 Howler 表现系统。

这些系统不是错，而是现在太早。当前缺的是构筑目标、推层目标和掉落反馈。如果现在把长期系统铺开，会稀释核心循环，增加调试和数值负担。

### 可以保留为数据预留

- `legendary`、`ancient` 品质类型。
- 强化石资源。
- `skillNodes` 类型。
- GSAP、Howler 依赖。

这些预留不需要删除，但不应成为下一阶段主线。

## 下一阶段设计主线

### 阶段 A：构筑评分可信

目标：玩家能选择一个评分偏好，并看到装备推荐随偏好变化。

验收：

- 暴击装在 `crit` 模式下更容易被判定为更优。
- 攻速装在 `speed` 模式下更容易被判定为更优。
- 生命护甲装在 `tank` 模式下更容易被判定为更优。
- 自动分解保护、只看更优、装备对比使用同一评分偏好。

### 阶段 B：推层目标清晰

目标：玩家知道当前适合挂机哪层、挑战哪层，以及为什么失败或收益降低。

验收：

- UI 展示推荐挂机层或当前收益建议。
- 战斗失败能说明主要差距。
- 推层成功有明确反馈。

### 阶段 C：掉落反馈有爽点

目标：关键奖励能被玩家感知，不只是日志文本变化。

验收：

- 传说装备有明显提示。
- 离线领取有完成反馈。
- 背包满和收益截断提示更明确。

## 产品风险

- 如果不做流派评分，装备系统会变成单一战力排序。
- 如果不做推层目标，玩家会不知道下一步为什么继续挂。
- 如果继续堆背包工具，背包整理会喧宾夺主。
- 如果先做长期系统，数值和 UI 复杂度会提前膨胀。

## 当前判断

当前方向没有跑偏，但已经到了需要收敛设计的时候。下一阶段应少做新系统，多做“让现有循环更有选择、更有目标、更有反馈”的工作。

# 放置型 ARPG 游戏设计文档（GDD）
> **文档版本**：v1.0  
> **游戏代号**：Idle ARPG  
> **目标平台**：H5（浏览器/微信/移动端 WebView）  
> **核心定位**：将传统 ARPG（暗黑破坏神、传奇、DNF）的"装备 Build 爽感"与放置挂机的"轻松节奏"相结合。玩家享受策略决策与角色成长，将重复的刷图劳动交给后台挂机系统。

---

## 1. 游戏概述

### 1.1 一句话描述
一款可以离线自动刷怪爆装备的放置型 ARPG，玩家回归后整理战利品、搭配装备与技能 Build、挑战更高层数，形成"越挂越强"的正向循环。

### 1.2 核心体验
- **离线即进度**：关闭游戏也是游戏的一部分，每次打开都有实质性成长。
- **决策重于操作**：玩家的价值体现在"选择穿什么、点什么天赋"，而非"手速多快"。
- **零挫败感**：没有"死亡惩罚"，打不过只是"收益降低"，不会丢失已有装备。
- **开箱爽感**：离线回归的战利品汇总，像开宝箱一样一件件展示极品装备。

### 1.3 目标用户画像
- 喜欢暗黑/传奇/DNF 的装备 Build 乐趣，但没时间/精力刷图的核心玩家。
- 偏好放置/挂机类游戏，希望有更深策略维度的休闲玩家。
- 追求"每天登录 2 次，每次 15 分钟"的轻量级游戏体验。

### 1.4 游戏时长设计
| 场景 | 时长 | 内容 |
|------|------|------|
| 单次整理 | 5-15 分钟 | 查看离线收益、筛选装备、调整 Build |
| 单次推层 | 2-5 分钟 | 手动挑战 Boss 层或验证新 Build |
| 日活跃 | 15-30 分钟 | 早晚各一次，无需连续在线 |
| 离线收益上限 | 8-12 小时 | 鼓励每天登录 1-2 次，避免焦虑 |

## 2. 核心游戏循环

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  挂机战斗   │────▶│  获取战利品 │────▶│  回城整理   │
│ (后台/离线) │     │ 装备/材料   │     │ 筛选/分解   │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                               │
                                               ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  验证战力   │◀────│  重新搭配   │◀────│  强化/合成  │
│  挑战推层   │     │  Build      │     │  提升品质   │
└──────┬──────┘     └─────────────┘     └─────────────┘
       │
       ▼
┌─────────────┐
│ 解锁更高层  │
│ 挂机收益提升│
└─────────────┘
       │
       └────────────────────────────────────────────┘
```

### 2.1 循环节点详解

**挂机战斗**
- 玩家设定当前挂机层数，系统自动进行战斗模拟。
- 战斗为纯数值计算，不需要玩家实时操作。
- 在线时可见战斗日志滚动和伤害飘字；离线时后台结算。

**获取战利品**
- 击杀怪物后根据掉落表生成装备、金币、强化材料。
- 装备品质从普通到传说随机 roll 点。
- 离线回归时以"战利品箱"形式一次性展示。

**回城整理**
- 筛选：设置自动拾取过滤规则（如"只捡稀有以上"）。
- 分解：白装/蓝装一键分解为强化石。
- 对比：新装备与身上装备并排对比，高亮更优属性。

**Build 搭配**
- 装备：根据词缀组合形成暴击流、攻速流、坦克流等。
- 技能：从天赋树中选择被动加成，与装备词缀联动。
- 属性：力量/敏捷/智力主副分，决定装备和天赋的加成方向。

**验证战力**
- 手动挑战新层数 Boss，验证 Build 有效性。
- 或设置"战力达标自动推层"，减少重复操作。

**解锁更高层**
- 新层数解锁更高 Item Level 的装备掉落。
- 传说装备掉率微量提升（主要靠后期系统调控）。

## 3. 系统详细设计

### 3.1 挂机战斗系统

**设计原则**：战斗不需要实时渲染，它是纯数值模拟。前端只需要"播放结果"。

**战斗单元**
- 以"秒"为单位进行批量计算，而非逐帧。
- 每秒计算：玩家对怪物造成的总伤害 vs 怪物对玩家造成的总伤害。
- 胜负判定：比较"玩家击杀怪物所需时间"与"怪物击杀玩家所需时间"。

**胜负结果**
- **胜利**：获得该怪物的经验、金币、掉落装备。进入下一只怪物战斗。
- **失败**：挂机暂停，自动回退到"已解锁的最高安全层数"。

**层数衰减机制**
- 每层有推荐战力。玩家战力低于推荐值时，收益按百分比衰减（而非直接禁止进入）。
  - 战力 ≥ 推荐：100% 收益
  - 战力 80%-100%：80% 收益
  - 战力 60%-80%：50% 收益
  - 战力 < 60%：20% 收益（勉强能挂）

**玩家可见反馈**
- 战斗日志滚动："你击败了 [怪物名]，获得 [装备名]"
- 实时伤害数字飘字（暴击时放大、变色、带拖尾）
- 当前层数、击杀速度、预计满包时间
- 血条与蓝条实时变化

### 3.2 装备生成系统

这是游戏的**核心发动机**，刷的动力全部来源于此。

#### 3.2.1 装备结构
```
装备 = 基础物品（Base Item）+ 品质（Rarity）+ 随机词缀（Affixes）
```

#### 3.2.2 品质层级

| 品质 | 词缀数量 | 特殊规则 | 视觉表现 | 掉落光柱 |
|------|---------|---------|---------|---------|
| 普通（Normal） | 0 | 仅基础属性 | 白色文字 | 无 |
| 魔法（Magic） | 1-2 | 随机词缀 | 蓝色文字，微弱光晕 | 无 |
| 稀有（Rare） | 3-4 | 可出现高级词缀 | 黄色文字，明显光晕 | 微弱光柱 |
| 传说（Legendary） | 4 | 1 个固定传奇词缀 + 3 个随机 | 橙色文字，独特模型/特效 | 明显光柱 |
| 远古/神话（Ancient） | 5 | 词缀数值范围更高 | 红色/紫色文字，全屏公告 | 全屏光柱 + 公告 |

#### 3.2.3 词缀（Affix）设计

分为**前缀（Prefix）**和**后缀（Suffix）**，每件装备有词缀数量上限。

**攻击类词缀**（适用：武器、戒指、手套）
- +X 攻击力
- +X% 暴击率
- +X% 暴击伤害
- +X% 攻击速度
- +X 火焰/冰霜/雷电伤害

**防御类词缀**（适用：护甲、头盔、鞋子、盾牌）
- +X 生命值
- +X 护甲值
- +X% 闪避率
- +X% 格挡率
- +X% 火焰/冰霜/雷电抗性

**功能类词缀**（适用：项链、戒指）
- +X% 经验获取
- +X% 魔法物品发现率（MF）
- +X% 金币获取
- +X% 移动速度

**传奇专属词缀**（仅传说装备拥有，固定出现）
- "暴击时 20% 几率冰冻敌人 2 秒"
- "击杀后爆炸，对周围敌人造成 50% 伤害"
- "攻击附加连锁闪电，弹射 3 个目标"
- "生命偷取：造成伤害的 5% 转为生命"

#### 3.2.4 装备部位划分

| 部位 | 数量 | 基础属性倾向 |
|------|------|-------------|
| 武器（主手） | 1 | 攻击力、攻击速度 |
| 副手 | 1 | 防御或法术加成 |
| 头盔 | 1 | 生命、抗性 |
| 护甲 | 1 | 护甲、生命 |
| 手套 | 1 | 攻击速度、暴击 |
| 鞋子 | 1 | 移动速度、闪避 |
| 戒指 | 2 | 功能词缀、攻击/防御混合 |
| 项链 | 1 | 功能词缀、全属性加成 |
| 腰带（后期扩展） | 1 | 生命、药水效果 |

#### 3.2.5 装备命名规则

根据词缀组合自动生成有代入感的名字：
- 魔法："[词缀名]的 [基础名]"（如"锋利的短剑"）
- 稀有："[前缀] [基础名] of [后缀]"（如"贪婪之钢铁长剑 of 烈焰"）
- 传说：固定独特名称（如"灰烬使者"、"影之哀伤"）

### 3.3 背包与筛选系统

挂机 8 小时后，背包可能有数百件装备。整理必须**高效、不烦躁**。

#### 3.3.1 核心功能

**自动拾取过滤**
- 玩家可设置规则："只拾取 Rare 以上"、"只拾取武器/护甲"、"只拾取包含 [暴击] 词缀的装备"。
- 不符合条件的装备直接转化为金币（而非进入背包）。

**一键分解**
- 白装/蓝装自动转化为强化石。
- 可设置保留规则："保留有 [XX 词缀] 的蓝装"。
- 分解前显示预计获得材料数量，确认后执行。

**装备对比**
- 鼠标悬停（或长按）新装备时，并排显示与身上同部位装备的属性差值。
- 用颜色标识：+12 攻击 ↑（绿色），-5% 暴击 ↓（红色）。
- 显示综合评分变化："替换后战力 +8%"

**智能高亮**
- 新掉落的装备如果综合评分高于身上同部位，自动标红/闪烁提示。
- 传说装备掉落时，全屏光柱 + 公告横幅（即使离线，回归后也在战利品箱中特殊标识）。

**套装检测**
- 穿戴界面实时显示"XX 套装 2/4 件已激活"。
- 未激活的套装部件在背包中用特殊边框标识。

### 3.4 Build 搭配系统

这是玩家的**策略核心**，也是游戏寿命的保障。

#### 3.4.1 三个维度

**1. 装备 Build（流派）**

| 流派 | 核心词缀 | 打法特点 |
|------|---------|---------|
| 暴击流 | 高暴击率 + 高暴击伤害 | 一击必杀，适合 Boss 战 |
| 攻速流 | 极限攻击速度 + 击中触发 | 高频触发特效，适合清怪 |
| 坦克流 | 高生命 + 高抗性 + 反伤 | 站撸不死，挂机稳定 |
| 元素流 | 单一元素伤害堆叠 + 元素穿透 | 针对特定抗性弱的怪物 |
| 召唤流（后期） | 召唤物数量 + 召唤物伤害 | 以多打少，适合挂机 |

**2. 技能与天赋树**

- **主动技能**：3-4 个（玩家可手动在 Boss 战释放，或设置为自动释放逻辑）。
- **被动天赋**：多分支选择，每次升级获得天赋点。
- **联动设计**：装备词缀与天赋互相放大。例如：
  - 装备词缀"冰冻敌人时伤害 +30%" → 天赋"攻击附带冰属性"
  - 天赋"暴击时 20% 几率重置技能冷却" → 装备堆暴击率

**3. 属性主副分**

| 属性 | 影响 | 适合流派 |
|------|------|---------|
| 力量（STR） | 物理攻击、护甲 | 战士、坦克 |
| 敏捷（DEX） | 暴击率、闪避 | 刺客、游侠 |
| 智力（INT） | 法术攻击、抗性、法力 | 法师、术士 |

玩家选择主属性后，装备和天赋围绕其构建。主属性提供 100% 加成，副属性提供 50% 加成。

#### 3.4.2 Build 验证器

- **实时面板**：显示当前 DPS、EHP（有效生命值）、期望击杀时间。
- **模拟对战**：选择某一层怪物，预测胜率与战斗时长。
- **战力评分**：综合评分变化提示（+15% ↑），让玩家"感觉"变强了。

### 3.5 离线收益系统

放置游戏的灵魂。

#### 3.5.1 计算逻辑

1. 记录玩家上次退出时间戳。
2. 再次登录时，计算离线秒数。
3. 设置离线收益上限（8-12 小时），超过部分不计入（防止无限膨胀也防作弊）。
4. 根据玩家离线前的"当前层数"和"Build"，模拟这段时间的战斗与掉落。
5. 汇总为一份"离线报告"。

#### 3.5.2 离线报告内容

```
┌─────────────────────────────────────────┐
│  欢迎回来！你离线了 7 小时 32 分钟       │
├─────────────────────────────────────────┤
│  击杀怪物：1,240 只                      │
│  获得金币：45,600                        │
│  获得装备：23 件                         │
│    ├─ 传说装备：1 件  [点击查看]         │
│    ├─ 稀有装备：5 件                     │
│    └─ 魔法装备：17 件                   │
│  强化材料：120 个                        │
│                                         │
│  [背包已满，提前 32 分钟停止收益]        │
│                                         │
│  [一键领取全部]                          │
└─────────────────────────────────────────┘
```

#### 3.5.3 呈现方式

- **开箱仪式**：不要直接发奖励，而是做一个"打开战利品箱"的动画，一件件展示，增强仪式感。
- **传说装备优先**：如果有传说装备，第一件展示它，配全屏光柱特效。
- **满包提示**：如果离线期间背包满了，明确提示"背包已满，损失 XX 时间收益"，促使玩家整理。

### 3.6 进度与层数系统

#### 3.6.1 层数结构

- **普通层**：每 10 只普通怪为一波，全自动挂机，无需干预。
- **Boss 层**：每 10 层一个 Boss（即第 10、20、30...层），可设计为必须手动点击释放技能（增加一点参与感，但非必须）。
- **层主怪物**：每层最后一只怪物稍强，掉落率微量提升。

#### 3.6.2 解锁机制

**手动验证制（默认）**
- 首次进入新层需要手动"挑战"并击败层主 Boss，之后该层开放挂机。
- 给玩家"征服感"："我亲手打过了这一层"。

**战力达标制（可选开关）**
- 当玩家战力超过该层推荐战力 120% 时，自动解锁。
- 减少后期重复操作，适合已成型 Build 的玩家。

#### 3.6.3 层数收益曲线

- **怪物强度**：每层怪物血量/攻击 = 上一层 × 1.08~1.12（指数增长）。
- **掉落 Item Level**：层数 ÷ 2 + 10（例如第 50 层掉落 iLv 35 的装备）。
- **传说掉率**：基础 0.1%，每 10 层 +0.05%，到达一定层数后通过其他系统调控。
- **金币收益**：与层数成正比，但受玩家金币获取词缀加成。

## 4. 数值设计

### 4.1 核心公式

#### 4.1.1 基础 DPS 计算
```
DPS = 基础攻击 × (1 + 主属性加成) × 攻击速度 × [1 + 暴击率 × (暴击伤害 - 1)]

其中：
- 基础攻击 = 武器基础攻击 + 装备词缀攻击总和
- 主属性加成 = 主属性值 × 0.5%（每点主属性 +0.5% 攻击）
- 攻击速度：基础 1.0 次/秒，上限 5.0 次/秒（软上限）
- 暴击率：上限 75%（软上限）
- 暴击伤害：基础 150%，即 1.5 倍
```

#### 4.1.2 有效生命值（EHP）
```
EHP = 生命值 / [(1 - 护甲减伤) × (1 - 抗性减伤) × (1 - 闪避率)]

其中：
- 护甲减伤 = 护甲值 / (护甲值 + 怪物等级 × 50 + 500)
- 抗性减伤 = 对应抗性 / (抗性 + 100)（每种元素单独计算，取最高伤害元素）
- 闪避率：上限 60%（硬上限，避免无敌）
```

#### 4.1.3 装备评分（Gear Score）
```
Score = Σ(词缀价值 × 权重系数)

权重根据当前 Build 的主属性动态调整：
- 暴击流：暴击率权重 2.0，暴击伤害权重 1.5，攻击权重 1.0
- 坦克流：生命权重 2.0，护甲权重 1.5，抗性权重 1.0
- 通用权重：主属性 1.0，攻击 1.0，防御 0.8，功能 0.5
```

### 4.2 数值平衡原则

#### 4.2.1 指数增长，但受控
- 怪物血量/攻击随层数指数增长（~1.1^x）。
- 玩家通过装备和天赋也指数增长，但两者之间的"差值"是游戏难度的来源。
- 玩家战力应始终略低于当前最高可挂层数的推荐战力，形成"刚好能过"的紧张感。

#### 4.2.2 软上限（Soft Cap）
- 任何单一词缀设置上限，迫使玩家多维发展：
  - 暴击率：75%
  - 攻击速度：500%
  - 闪避率：60%（硬上限）
  - 单元素抗性：80%

#### 4.2.3 边际递减
- 同一词缀堆得越多，每件装备提供的相对收益越低。
- 例如：从 0% 到 50% 暴击率，每 10% 提升巨大；从 60% 到 70%，提升感知明显减弱。

#### 4.2.4 可见的进步
- 每次 Build 调整后，必须给玩家一个"战力评分"的变化提示（+15% ↑）。
- 即使实际层数只推了 2 层，也要让玩家"感觉"变强了。

### 4.3 经济系统

#### 4.3.1 货币类型（前期极简）

| 货币 | 用途 | 获取方式 | 消耗速度 |
|------|------|---------|---------|
| **金币** | 基础强化、修理、购买药水 | 挂机掉落、分解装备 | 中等 |
| **强化石** | 装备强化 +1~+10 | 分解白/蓝/黄装 | 快（后期大量需求） |
| **远古精华**（后期） | 传说装备重 roll 词缀、远古升级 | 分解传说装备、高层 Boss | 极慢 |

**设计原则**：前期只有金币 + 强化石两种货币，避免"货币膨胀导致玩家失忆"。远古精华等高级货币在玩家首次获得传说装备后引入。

#### 4.3.2 强化系统

| 强化等级 | 成功率 | 失败惩罚 | 视觉反馈 |
|---------|--------|---------|---------|
| +1 ~ +3 | 100% | 无 | 微弱白光 |
| +4 ~ +6 | 80% | 无 | 蓝色光晕 |
| +7 ~ +9 | 60% | 降 1 级 | 紫色光晕 + 特效 |
| +10 | 40% | 不变（保护） | 金色全屏闪光 |
| +11+（后期） | 30% | 破碎风险 | 红色警告光效 |

**强化规则**：
- 强化只提升装备"基础属性"（白字），不影响词缀。
- 每次强化消耗金币 + 强化石，等级越高消耗越多。
- +7、+9、+10 为关键节点，应有特殊视觉和音效反馈（参考传奇武器发光）。

## 5. 视觉与音效设计

### 5.1 视觉风格

- **整体基调**：暗黑奇幻，但 UI 保持清晰可读。参考暗黑 3 的暗色背景 + 高饱和度装备光效。
- **色彩编码**：严格遵循品质颜色体系（白→蓝→黄→橙→红），玩家一眼识别价值。
- **布局**：左侧角色面板（属性/装备），中间战斗区域（日志/飘字），右侧背包/技能。

### 5.2 关键视觉反馈

| 场景 | 视觉表现 | 音效 |
|------|---------|------|
| **普通掉落** | 装备图标从怪物位置飞入背包 | 轻微"叮"声 |
| **稀有掉落** | 黄色光柱从地面升起，装备带光晕 | 清脆金属声 |
| **传说掉落** | 全屏橙色光柱，底部公告横幅，装备模型旋转展示 | 史诗级号角声 + 低音震动 |
| **离线回归** | 战利品箱开箱动画，逐件展示，传说优先 | 开箱金属声 + 每件掉落对应音效 |
| **强化成功** | +7 蓝色闪光、+9 紫色闪电、+10 金色全屏爆发 | 对应等级的强化音效，+10 配欢呼声 |
| **暴击伤害** | 伤害数字放大 1.5x、红色、带拖尾残影 | 重击声 |
| **连击击杀** | 数字合并显示 "×5 连杀！"，屏幕微震 | 连续击杀 escalating 音效 |
| **层数突破** | 屏幕中央 "第 XX 层 突破！"，烟花粒子 | 胜利号角 |

### 5.3 UI 设计原则

- **信息密度**：ARPG 玩家习惯高密度信息，但不要一次性展示所有。用折叠面板、悬停提示分层。
- **一键操作**：任何需要重复点击的操作都必须有一键替代（一键分解、一键穿戴最优、一键推层）。
- **对比清晰**：装备对比时，属性差异用颜色 + 箭头 + 百分比，3 秒内看懂。
- **暗黑模式**：默认暗黑主题，减少眼睛疲劳（玩家可能深夜整理装备）。

## 6. 游戏进度与 Pacing

### 6.1 新手期（0-2 小时）

**目标**：建立"掉装备 = 变强"的核心爽感。

- **前 10 分钟**：手动点击杀怪，每 2-3 场战斗掉落一件更好的装备，频繁换装。
- **前 30 分钟**：解锁自动挂机，玩家看到角色自己打怪、自己捡装备。
- **前 2 小时**：解锁离线收益、背包、分解、第一层 Boss。玩家知道"以后可以轻松玩"。

**关键设计**：前期掉率人为调高，确保玩家快速体验全套品质层级。

### 6.2 成长期（第 1-7 天）

**目标**：形成初步 Build，推到第 50-100 层。

- **第 1 天**：推到第 10-20 层，获得第一件传说装备（或至少看到掉落）。
- **第 3 天**：围绕某个流派（暴击/攻速/坦克）形成 4-6 件核心装备。
- **第 7 天**：首次击败第 50 层 Boss，解锁"远古装备"概念。

**关键设计**：第 1 天必须给传说装备（或极高期待），否则玩家流失。

### 6.3 深化期（第 2-4 周）

**目标**：优化 Build，追求极品词缀，挑战高层。

- 玩家开始追求"完美词缀组合"（如"暴击率 + 暴击伤害 + 攻击速度 + 主属性"四词缀稀有）。
- 强化系统成为消耗出口，+7 以上开始有策略性（是否值得强化这件装备？）。
- 解锁转生/轮回系统（Prestige），到达一定层数后可重置获得永久加成。

### 6.4 终局期（1 月+）

**目标**：长期追求，社交展示。

- 多角色/多职业系统（战士、刺客、法师），共享部分仓库和加成。
- 排行榜：层数排行、最快推层记录、最高战力评分。
- 装备展示：玩家将极品装备贴到公共展示区（类似 DNF 装备截图）。
- 限时挑战：每周特殊规则挑战（如"只用蓝装通关第 100 层"）。

## 7. 参考游戏分析

### 7.1 直接参考（放置/挂机类）

**Melvor Idle**
- **借鉴**：完整的 RPG 系统（战斗、技能、装备、银行）全部挂机化，证明 ARPG 可以 Idle。
- **规避**：UI 过于朴素，我们的装备光效和开箱感需要更强。

**Clicker Heroes**
- **借鉴**：层数推进 + 离线收益 + 转生系统（Prestige）的数值框架。
- **规避**：后期数值膨胀极快，需设计软上限和多元追求（不只是层数）。

**各种"放置传奇"类手游**
- **借鉴**：传奇的装备光效、自动战斗、离线时长设计。
- **规避**：避免付费逼氪，离线时长可通过游戏内成就解锁，保持公平性。

### 7.2 间接参考（传统 ARPG）

**暗黑破坏神 2/3/4**
- **借鉴**：装备词条池设计、套装效果、词缀 Tier、传奇光柱。
- **转化**：将"手动刷图 100 小时"转化为"挂机 100 小时 + 回来整理 10 分钟"。

**地下城与勇士（DNF）**
- **借鉴**：装备部位划分、强化/增幅系统、技能树分支、转职概念。
- **转化**：将"深渊票刷图"转化为"层数挂机 + 体力系统"。

**传奇/奇迹 MU**
- **借鉴**：光效品质体系（白→绿→蓝→紫→橙→红）、+7/+9 发光、PK 掉落刺激感。
- **转化**：将"通宵挂机练级"转化为"离线自动收益 + 回来开箱"。

**流放之路（PoE）**
- **借鉴**：复杂的 Build 可能性（天赋星盘、技能石联动）。
- **转化**：只取其"深度"，大幅简化"复杂度"（我们的天赋树控制在 30-50 个节点）。

## 8. 风险与规避

| 风险 | 影响 | 规避方案 |
|------|------|---------|
| **前期掉率过高导致后期乏味** | 玩家快速毕业，失去追求 | 后期引入"远古装备"、"词缀重 roll"、"套装收集"等深层追求 |
| **数值膨胀失控** | 战力差距过大，新系统无法兼容 | 所有数值用公式驱动，定期审计；引入软上限 |
| **离线收益被破解** | 修改本地时间获取无限收益 | 服务端校验（如有服务器）或收益上限硬封顶 |
| **Build 单一化** | 某个流派过强，其他流派无人问津 | 定期平衡调整；Boss 层设计抗性轮换，迫使多流派准备 |
| **整理疲劳** | 装备太多，筛选耗时过长 | 强大的自动过滤 + 智能高亮 + 一键分解 + 装备评分辅助决策 |

## 9. 术语表

| 术语 | 英文 | 解释 |
|------|------|------|
| Build | Build | 角色构建，装备+技能+天赋的综合搭配方案 |
| 词缀 | Affix | 装备上的随机属性条目，分前缀和后缀 |
| 前缀 | Prefix | 通常偏向攻击/防御的词缀 |
| 后缀 | Suffix | 通常偏向功能的词缀 |
| 装备等级 | Item Level (iLv) | 决定词缀的数值范围和可出现的词缀池 |
| 有效生命值 | EHP | Effective HP，综合护甲、抗性、闪避后的实际生存能力 |
| 每秒伤害 | DPS | Damage Per Second |
| 转生 | Prestige | 到达一定进度后重置游戏，获得永久加成，重新开始但更快 |
| 软上限 | Soft Cap | 超过后收益大幅降低但仍有微量提升 |
| 硬上限 | Hard Cap | 绝对无法超过的上限 |
| 魔法物品发现率 | MF | Magic Find，提升高品质装备掉率的功能词缀 |
| 对象池 | Object Pool | 前端性能优化手段，复用对象而非频繁创建销毁 |
